# page-task-intake

This is Step 1 of the Product-Led Page Design workflow.

It converts a raw or incomplete page design request into a structured Page Task Definition.

## Folder contents

- `SKILL.md`: Main Claude Skill instructions
- `principles.md`: Product/design principles for this step
- `templates/input-schema.yaml`: Expected input shape
- `templates/output-schema.yaml`: Stable output schema
- `templates/page-task-definition.md`: Human-readable output template
- `checklists/quality-check.md`: Validation checklist
- `examples/minimal-input.md`: Minimal example input
- `examples/complete-output.md`: Example output
- `tests/test-cases.md`: Test cases for skill behavior

## Intended use

Use before any page strategy, content, wireframe, UI, or designer handoff work.

This skill is intentionally narrow so it can be tested and reused independently.
