# Page Task Intake Quality Checklist

Use this checklist before completing the skill output.

## Scope control

- [ ] The output defines the page task only.
- [ ] The output does not generate page sections.
- [ ] The output does not write final marketing copy.
- [ ] The output does not create a wireframe.
- [ ] The output does not propose UI visual directions.
- [ ] The output does not produce a designer handoff brief.

## Required fields

- [ ] `page_type` is present or marked `unknown`.
- [ ] `target_user` is present or marked `unknown`.
- [ ] `primary_business_goal` is present or marked `unknown`.
- [ ] `desired_user_action` is present or marked `unknown`.
- [ ] `initial_page_mission` is present, even if provisional.
- [ ] `unknowns` is present.
- [ ] `missing_information` is present.
- [ ] `next_step_readiness` is present.

## Fact handling

- [ ] Confirmed facts are separated from assumptions.
- [ ] No unsupported details are presented as facts.
- [ ] Proposed metrics are marked as assumptions if not provided by the user.
- [ ] Derived page names are marked as assumptions if not provided.

## Readiness

- [ ] If the four minimum fields are clear, status is `ready`.
- [ ] If the task is understandable but incomplete, status is `partially_ready`.
- [ ] If the task cannot be defined, status is `blocked`.
- [ ] For blocked cases, no more than 5 intake questions are asked.

## Downstream usability

- [ ] The output can be used directly by the Business Context Structuring skill.
- [ ] The structure is stable and machine-readable.
- [ ] The language is concise and unambiguous.
