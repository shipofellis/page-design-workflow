
# Orchestration Frameworks

## Workflow State Framework

Possible states:

- intake
- context-building
- gap-analysis
- discovery
- strategy
- content-structuring
- wireframing
- expert-review
- cross-functional-alignment
- handoff
- retrospective
- blocked
- revision-loop

## Readiness Framework

### Ready
Safe to advance.

### Partially Ready
Can continue with explicit assumptions, accepted risks, or owned open items.

### Blocked
Critical issue prevents progression.

## Alignment Requirement Framework

Cross-functional alignment is required when:

- concerns affect more than one discipline
- implementation or architecture risk is material
- PM/design/engineering priorities conflict
- handoff would carry unresolved tradeoffs
- ownership for open issues is unclear

## Revision Loop Framework

Rollback triggers include:

- unresolved contradictions
- severe UX risks
- weak conversion logic
- unstable strategy
- missing trust logic
- implementation infeasibility
- alignment failure

## Artifact Dependency Framework

Each step depends on upstream artifacts.

Never advance without:
- dependency validation
- readiness validation
- assumption preservation
- alignment validation when applicable

## Assumption Lifecycle Framework

States:
- created
- carried-forward
- validated
- invalidated
- retired

## Decision Lifecycle Framework

States:
- proposed
- agreed
- rejected
- deferred
- escalated
- accepted-risk
