
# Cross-Functional Alignment Checklist

## Input Readiness
- Page Strategy Document exists
- Content-First Page Draft exists
- Wireframe Specification exists
- Multi-Role Review Report exists

## Concern Consolidation
- duplicate concerns merged
- severity assigned
- affected artifact identified
- source roles preserved

## Conflict Quality
- conflicts named explicitly
- tradeoffs described
- decision options listed
- escalation items identified

## Decision Quality

Every major decision includes:
- rationale
- alternatives considered
- accepted risk
- owner
- affected artifacts

## Implementation Alignment
- frontend constraints documented
- architecture concerns documented
- responsive constraints documented
- analytics or instrumentation needs captured

## Handoff Readiness
- fixed requirements clear
- designer decision space clear
- open items assigned
- blockers identified
- readiness status declared

## Output Validation
- no false consensus
- no unresolved blocker hidden
- no generic critique pass
- no final UI generated
- no unowned open questions
