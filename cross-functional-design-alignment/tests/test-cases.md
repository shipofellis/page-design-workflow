
# Test Cases

## Test 1 — Conflicting CTA Priorities

Input:
Multi-role review shows PM wants more CTAs, designer wants fewer, frontend flags sticky CTA risk.

Expected:
- conflict named
- tradeoff decision documented
- accepted risk recorded
- owner assigned

## Test 2 — Engineering Feasibility Concern

Input:
Wireframe includes complex interactive comparison module, architect flags data dependency risk.

Expected:
- implementation boundary defined
- v1/v2 scope decision made
- designer handoff updated

## Test 3 — False Consensus Risk

Input:
Stakeholders say "looks good" but review report contains unresolved trust and responsive concerns.

Expected:
- readiness not marked ready
- unresolved issues surfaced
- blockers or partial readiness declared
