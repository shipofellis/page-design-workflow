
# Gap Analysis Checklist

## Input Validation

- Page Task Definition exists
- Business Context Document exists
- Product overview exists
- Business goal exists
- Target user exists

---

## Gap Quality

Every gap:
- has category
- has impact explanation
- has severity
- has follow-up question
- avoids generic wording

---

## Strategic Coverage

The report evaluates:

- user understanding
- business goals
- conversion logic
- trust gaps
- messaging clarity
- positioning
- differentiation
- CTA readiness
- analytics readiness
- implementation constraints

---

## Output Validation

- no hallucinated information
- no fake certainty
- blocking issues explicit
- assumptions explicit
- next-step readiness included
