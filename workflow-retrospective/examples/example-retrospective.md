
# Example Retrospective Excerpt

## What Failed

Issue:
The designer had to ask for the primary user objection during visual exploration.

Evidence:
Design review notes showed repeated questions about why users would hesitate before signup.

Impact:
Trust section design was delayed and initially underdeveloped.

Root Cause:
Step 3 gap analysis did not require identification of primary conversion objection.

Related Workflow Step:
page-requirement-gap-analysis

Recommended Fix:
Add a required gap check:
"Is the user's strongest pre-conversion objection known?"

---

## SOP Update Recommendation

Affected Step:
Step 3 — Requirement Gap Analysis

Proposed Change:
Add primary conversion objection as a mandatory field in the gap checklist.

Evidence:
Missing objection data caused design review friction.

Expected Benefit:
Improves trust strategy and reduces designer clarification loops.

Priority:
High
