
# workflow-retrospective

This skill runs a structured retrospective after the product-led page design workflow has produced a handoff, design outcome, or launch feedback.

Primary objective:
turn completed workflow execution into reusable knowledge, validated patterns, and SOP improvements.

Recommended workflow:

1. page-task-intake
2. business-context-structuring
3. requirement-gap-analysis
4. requirement-discovery-qna
5. page-strategy-definition
6. content-first-page-draft
7. wireframe-specification
8. multi-role-review
9. designer-handoff-brief
10. workflow-retrospective

Best used after:
- designer handoff
- first design review
- implementation feedback
- launch
- experiment result
- stakeholder review
