
# Retrospective Frameworks

## Evidence Classification Framework

### Strong Evidence
- conversion metrics
- A/B tests
- user interviews
- usability testing
- behavioral analytics

### Medium Evidence
- designer feedback
- engineering feedback
- stakeholder review notes
- sales/customer success feedback

### Weak Evidence
- PM judgment
- isolated opinions
- untested assumptions
- aesthetic preference

---

## Learning Classification Framework

Classify each learning as:

- workflow learning
- user insight
- product strategy learning
- content learning
- UX learning
- design collaboration learning
- engineering learning
- analytics learning
- prompt improvement

---

## Assumption Status Framework

### Validated
Evidence supports the assumption.

### Invalidated
Evidence contradicts the assumption.

### Partially Validated
Evidence supports only part of the assumption.

### Unknown
No sufficient evidence exists.

---

## Reusable Pattern Framework

A reusable pattern must specify:

1. name
2. problem solved
3. required conditions
4. when to use
5. when not to use
6. known risks
7. example
8. related workflow step

---

## SOP Patch Framework

Each SOP patch should include:

1. affected step
2. current weakness
3. proposed change
4. evidence
5. expected impact
6. rollback condition
