
# Common Layout Frameworks

## Hero Layout

Common patterns:
- left copy / right product visual
- centered messaging
- split comparison layout

Choose based on:
- product complexity
- trust requirements
- visual proof needs

---

## Feature Grid Layout

Best for:
- scannability
- modular products
- multi-feature systems

Risks:
- flattening hierarchy
- repetitive cards

---

## Storytelling Layout

Best for:
- narrative progression
- onboarding concepts
- educational flows

Requires:
- strong transition logic

---

## Comparison Layout

Best for:
- differentiation
- competitive positioning
- pricing explanation

Requires:
- strong information prioritization

---

## Trust Layout

Possible elements:
- logo rows
- testimonials
- metrics
- security indicators

Trust placement should match user skepticism timing.

---

## CTA Reinforcement Layout

Purpose:
- consolidate confidence
- reduce friction
- simplify decisions

Should appear after:
sufficient trust and understanding exist.
